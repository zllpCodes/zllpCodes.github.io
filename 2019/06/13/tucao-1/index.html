工作中遇到的奇怪bug😂不吐不快(1)

2019-06-13

技术 吐槽

前情提要

工作嘛，总有一堆烦心事；每天就是努力扮演一个情绪稳定的成年人。
然后在遇到奇奇怪怪的bug之后，总能一秒破功，让人抓狂，尤其是当bug实在是太智障的时候，让人不禁想问一句：“这尼玛都什么沙雕玩意儿？！”

⚠️在此绝不diss任何码农、攻城狮、程序猿。生活不易，头顶易秃，和气生财。

使用webpack-api-mocker掉坑记

为了配置新项目的本地mock数据，之前的方式只能接收GET请求，不支持POST。但现在有POST的接口，那当然得适时的update一下配置。在网上搜到了 webpack-api-mocker，看了参考文档，感觉很不错，符合我的需求；在现有的webpack基础上新增配置简单，改动原有配置小，嗯，就是它了！kuakuakua，一通操作，运行起来，结果报错了。行吧，咱也不敢说啥，还能怎么样呢，生活天天对我这只小猫咪下手，debug呗。

吸取了之前的经验，我现在要认真的看看报错。

...
Module build failed:
  Error:
    You gave us a visitor for the node type "ReferencedIdentifier"
    but it's not a valid type
...

然后使用面向谷歌编程技能，搜出来这都是啥？没一个对得上的好吗？Exm？

不行，我是成年人了。我要冷静。我要冷静。我要冷静。

10min后。

喝着巧克力牛奶。啊，快乐果然是甜食给的。我又可以愉快的debug了。

既然报错没能够给值得参考的信息，那就只能认真研究文档了。

经过我的不断比对后，我发现：嗯？这个文件的路径好像不是很一样，但是这个应该没有关系吧？要不还是按照文档配置一下路径吧，万一呢？

然后我就尝试性的更改了文件路径，然后，就顺利运行了。Exm？？？

再然后我又买了一杯柠檬茶。啊，我爱甜食。

之前把文件放在根目录下，而不是新建了一个mocker文件夹放在里面。因为没有去看源码，所以其实根本没资格说沙雕。在此只是完全个人向的吐槽。

最后要谢谢各位维护代码的大佬们，因为有你们的付出才能让吾等小白更愉快的写代码。❤️ 爱你萌
